Purpose and Learning Focus

This formative assignment prepares you for the Summative11 Programming Fundamentals assignment by helping you understand how basic programming concepts appear in common business software environments, not how to become a programmer.

You will explore foundational programming ideas and reflect on:

  • how automation tools such as macros translate user actions into executable instructions,
  • how different programming environments respond to commands, inputs, and execution steps,
  • and how observable program behavior can serve as evidence that a task was completed correctly.

This is a low-stakes, preparatory assignment. You are expected to experiment, observe, and reason about outcomes-not write complex code. Grading emphasizes conceptual understanding and interpretation of results, not syntax mastery.

After completing this assignment, you should be able to:

  • recognize how simple programs and macros automate repetitive tasks in business applications,
  • explain why programming tools often require more steps than expected to perform simple actions,
  • describe how command-line and interactive environments provide immediate feedback during execution,
  • interpret visible program output as evidence of correct logic or execution flow,
  • and prepare to evaluate and document program behavior in the companion Summative assignment.

You are not expected to master programming languages. Focus on understanding what programs do, how users interact with them, and how outcomes can be verified.
Advanced coding topics are intentionally out of scope for this formative exercise.

Exercise 1-Concepts

ref: MIS202 LinkedIn Learning

For this Exercise listen to the LinkedIn Learning course: 'Excel Macros and VBA for Beginners ; ' and answer the following.

(10) 1. Why do professionals choose to automate repetitive Excel tasks using macros instead of performing the same steps manually each time?
Focus on efficiency, consistency, and risk reduction in business workflows.


(10) 2. In a business setting, what types of Excel tasks are most appropriate for macro automation, and why?
Think about frequency, predictability, and the consequences of human error.

(10) 3. How does recording a macro help non-programmers understand what a program does without writing code from scratch?
Focus on observation of actions, inputs, and outcomes rather than syntax.

(10) 4. If a manager or instructor could only see the final result of a macro running in Excel, what visible outcomes would indicate that the automation worked correctly?
Do not list steps-describe observable results and behavior.
